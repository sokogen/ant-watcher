GitHub Actions Webhooks Knowledge Base
Last updated: 2024-10-21
GitHub Actions version: Current as of April 2024
Overview
This document describes the structure and content of webhooks sent by GitHub Actions server.
Event Types and Actions

workflow_run

requested
in_progress
completed


workflow_job

queued
in_progress
completed


check_run

created
completed
rerequested
requested_action


check_suite

completed
requested
rerequested



Payload Structure
Common fields:

action
repository
sender

Specific fields:

workflow_run: workflow_run, workflow
workflow_job: workflow_job
check_run: check_run
check_suite: check_suite

Important Payload Fields

run_id: Unique identifier for the workflow run
status: Current status of the run or job
conclusion: Final result (for completed events)
created_at, started_at, completed_at: Timestamps
head_sha: Commit SHA that triggered the workflow
steps: Detailed information about job steps (workflow_job only)
check_run_url: URL for the check run (workflow_job only)
html_url: URL to view the run or job in the GitHub UI
pull_requests: Array of related pull requests (if applicable)
event: Triggering event (e.g., "push", "pull_request")

Event Sequence

workflow_run.requested
workflow_job.queued
workflow_run.in_progress
workflow_job.in_progress
workflow_run.completed
workflow_job.completed

Note: check_run and check_suite events may occur at various points during this sequence.
Request Headers

X-GitHub-Event: Event type (workflow_run, workflow_job, check_run, or check_suite)
X-GitHub-Delivery: Unique delivery identifier
X-GitHub-Hook-ID: ID of the webhook that triggered the delivery
X-GitHub-Hook-Installation-Target-ID: ID of the installation that the event relates to
X-GitHub-Hook-Installation-Target-Type: Type of the installation target
Content-Type: application/x-www-form-urlencoded
User-Agent: GitHub-Hookshot/<SHA>
Accept: /
